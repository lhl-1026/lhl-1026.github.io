1.前端代理配置

a..env， .env.development， .env.production的区别及使用场景

1..env：

用途：这是全局默认的配置文件，无论在哪个环境下运行Vue项目，都会加载这个文件中的环境变量。

使用场景：通常用于存储项目的基本配置，如应用的名称、版本信息等，这些配置在所有环境中都是相同的

2..env.development：

用途：这是专门为开发环境设计的配置文件。当在开发模式下运行Vue项目时（例如，通过npm run serve或yarn serve命令），会自动加载这个文件中的环境变量。

使用场景：开发环境通常需要一些特定的配置，比如连接到开发用的数据库、API接口、开启调试模式等。这些配置通常不应该在生产环境中使用，因此会放在.env.development中。

3..env.production：

用途：这是专门为生产环境设计的配置文件。当构建并部署Vue项目到生产环境时（例如，通过npm run build或yarn build命令），会自动加载这个文件中的环境变量。

使用场景：生产环境通常需要保密和优化的配置，比如连接到生产用的数据库、API接口、关闭调试模式、设置性能优化选项等。这些配置是确保应用在生产环境中稳定运行的关键。
